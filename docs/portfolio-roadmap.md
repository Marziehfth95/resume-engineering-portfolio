# Portfolio Roadmap

## Vision

Ein Repository, das nicht nur Code speichert, sondern eine glaubwuerdige Story erzaehlt:
Ich lerne aktiv, baue konsequent und kann Projekte von der Idee bis zur Umsetzung strukturieren.

## Phase 1

- Grundstruktur aufsetzen
- README fuer alle Hauptbereiche anlegen
- Erste Trainingsaufgaben dokumentieren

## Phase 2

- Mindestens ein vollstaendiges Backend-Projekt umsetzen
- Mindestens ein visuell starkes Frontend-Projekt praesentieren
- Relevante Screenshots und Demo-Links einpflegen

## Phase 3

- Testing und CI ergaenzen
- Projektmetriken und technische Entscheidungen dokumentieren
- Repository fuer Bewerbungen feinpolieren

## Quality Standard

Jeder Bereich sollte folgende Fragen beantworten:

- Was wurde gebaut oder geuebt?
- Warum ist es relevant?
- Welche Technologien kamen zum Einsatz?
- Was habe ich dabei gelernt?
